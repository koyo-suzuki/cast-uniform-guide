# キャスト制服制度 Webページ 情報設計（IA）v3

- 対象読者: キャスト（スマホ閲覧が主と想定）
- スコープ: 制服制度のみ（表彰式全体は含めない）
- 形式: 単一HTMLページ（アンカーナビ付きロングページ）を推奨
- 更新日: 2026-07-21（v3: 制度比較表・適用開始日・更新日を追加）

## 1. 設計の前提（重要）

2つの制度は**別軸の制度であり、比較・選択されるものではない**。動機がそもそも違う。

| 制度 | 応える動機 |
|---|---|
| **マイ制服買取（購入型）** | 「他の子と同じ制服を共有したくない」「衛生面が気になる」→ 既存デザインをもとに新規制作し、本人専用にする |
| **オリジナル制服（達成型）** | 「自分だけのデザインの制服が欲しい」→ 新規デザインを制作してもらう |

よってページの核は比較表ではなく、**動機ベースの入口（目的別ナビ）**。「あなたはどっち?」と選ばせるのではなく「こういう子はこの制度」と案内する。

## 2. ソース

| ソース | 使う内容 |
|---|---|
| スプレッドシート「キャスト表彰式」→ オリジナル制服シート | 達成型制度の条件・失効条件・所有権/著作権・申請条件 |
| マイ制服買取制度_制度設計ドラフト.md（2026-06-02 経営会議） | 購入型制度の骨子・価格・管理・キャンセル不可・命名整理 |

## 3. ページ構造（推奨: 1ページ構成）

```
キャスト制服制度ガイド
├─ ① ヒーロー: 「制服の制度が新しくなりました」+ 貸し借り全面禁止の告知
├─ ② 目的別ナビ（2枚のカード・このページの入口）
│   ├─ カードA「自分だけのデザインを作りたい」→ オリジナル制服セクションへ
│   ├─ カードB「自分専用の制服で働きたい」→ マイ制服買取セクションへ
│   └─ 2制度の比較表（目的・費用・所有者・条件・受け取りまで）
├─ ③ オリジナル制服（達成型）詳細
│   ├─ こんな子のための制度（自分のデザインを形にしたい 等）
│   ├─ 取得条件（キャスト / キャストMGRでタブ切替）
│   ├─ 失効条件（勤怠・契約・制作・制度ルールの4分類アコーディオン）
│   ├─ 所有権・著作権・修正費用
│   └─ 申請手順（ステップ表示）
├─ ④ マイ制服買取（購入型）詳細
│   ├─ こんな子のための制度（共有が苦手・衛生面が気になる 等）
│   ├─ 制度の仕組み（既存デザインをもとに新規制作し、本人専用で着用）
│   ├─ 価格・支払い方法（会社口座へ振込）
│   ├─ 製作期間 3〜6ヶ月 / キャンセル不可（警告ボックスで強調）
│   └─ 申込手順（フォームへのリンク）
├─ ⑤ 共通ルール
│   ├─ 貸し借り全面禁止（両制度の大前提・最上部①でも告知）
│   ├─ 管理方法（専用袋・ロッカー / 洗濯・保管ルール）
│   └─ 風営法関連の禁止事項（煽り行為等）
├─ ⑥ FAQ（アコーディオン）
└─ ⑦ 問い合わせ先・申請導線（固定フッターCTA: 制度別2ボタン）
```

### 設計判断
- 冒頭に**概要比較表を置く**。優劣ではなく、目的・費用・所有者・条件・受け取りまでの違いを短く確認できる内容にする。
- 順序はオリジナル制服を先にする（2026-07-21 Koyo確認）。
- ②の目的別ナビは「悩み・気持ち」の言葉で書く（制度名で書かない）。例:「みんなで共有する制服に抵抗がある…」「自分の好きなデザインで働きたい!」
- 両制度は併用可否が論点になり得る → FAQで明示（要確定）。

## 4. セクション別の中身

### ③ オリジナル制服
- 冒頭リード文「世界に一着、自分がデザインした制服を作れる制度」
- 条件集計前の事前申請が必須であることを、見出し直下の注意枠で明示
- 取得条件はキャスト/キャストMGRで数値が違うため**タブまたは2カラム**（売上150万/100万、出勤30日/店責10日、3ヶ月連続/3回達成、予算 税込10万円、年間1着まで）
- 失効条件4分類は長文のため**アコーディオン**必須。在籍条件には退店・長期休暇に加え、退店意思を示して契約協議・退店日調整中のケースも含める
- 申請手順: KM承認 → イントラ「オリジナル制服申請」を期間7日前までに、をステップUIで

### ④ マイ制服買取
- 冒頭に「こんな子のための制度です」リード文（潔癖・共有への抵抗を、角が立たない表現に言い換える。例:「自分だけが袖を通す制服で働きたい子へ」）
- **既存デザインをもとに新規制作し、所有者は店舗・着用者は申込者本人**という整理を見出し直下と図解で明示
- 「キャンセル不可」「製作3〜6ヶ月」はドラフトで明示必須とされている → 警告ボックス
- 申込フォームは未作成（アクションアイテム）→ リンク枠だけ用意

### ⑥ FAQ 候補
- マイ制服買取とオリジナル制服、両方やってもいい? → **併用可で確定（2026-06-12）**
- すでに「専用制服」を持っている場合は? → 旧「専用制服」は現「オリジナル制服」であることを案内
- 退職したら制服はどうなる?（達成型: 2年所有権ルール / 購入型: 未確定）
- サイズが変わったら?（修正費用は自己負担）

## 5. UI/UX方針

- スマホファースト。カード/アコーディオン中心の縦積み
- 数値（150万・10万・12〜13万・3〜6ヶ月）はビジュアルで大きく
- 未確定項目は「※確定後更新」ラベルを統一デザインで付け、確定前公開でも破綻しない作りに
- 固定フッターCTA: 「マイ制服買取を申し込む」「オリジナル制服を申請する」の2ボタン

## 6. 公開前に確定が必要な事項（ブロッカー）

1. **新制度の正式名称**: マイ制服買取で確定（2026-07-16）
2. 価格 12〜13万円の確定
3. 対象デザインの範囲（全店舗か特定店舗か）
4. 占有権の有効期間・退職時の扱い
5. 旧「専用制服」保持者への名称変更の周知（FAQ反映済み）
6. 洗濯・保管ルールの具体内容
7. 申込フォームのURL・項目

※両制度の併用可否は**併用可で確定**（2026-06-12 Koyo確認）→ 解消済み
